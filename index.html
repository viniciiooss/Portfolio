Projetos em Destaque

Pipeline PPE (Diários Oficiais)

Sistema automatizado para processar o Diário Oficial do Estado do Piauí, extraindo e classificando movimentações de servidores públicos a partir de PDFs não estruturados até a entrega em Excel e banco relacional.

FastAPI Agno Agents OCR via LLM Docker MySQL

O PPE (Promoções, Progressões e Enquadramentos) automatiza o processamento do Diário Oficial do Estado do Piauí para identificar promoções, progressões, enquadramentos e nomeações de servidores. O pipeline transforma PDFs complexos e pouco estruturados em registros organizados para consumo operacional e gerencial.

Arquitetura do pipeline

  • Entrada documental: PDFs do DOEPI processados com poppler-utils e tesseract-ocr para OCR e extração inicial.
  • Camada inteligente: agentes LLM gerenciados pelo framework Agno, com Gemini 2.5 Flash/Pro como principal e Kimi como fallback automático.
  • Automação operacional: interface em PHP CodeIgniter para disparo e monitoramento dos jobs, com execução diária orquestrada por cron/n8n às 23:59.
  • Saída estruturada: persistência em MySQL remoto e exportação final em `.xlsx` para uso direto por equipes gestoras.
  • Configuração parametrizada: padrões de busca, modelos, datas, limites e labels definidos em YAML editável sem alterar o código.

Confiabilidade e segurança

  • Autenticação JWT com renovação automática e controle de acesso por papéis.
  • Rate limiting, audit logging e deploy com Docker Compose em workspaces separados.

Plataforma Macroeconômica com LLM

Plataforma completa para ingestão, persistência e exploração de indicadores econômicos brasileiros com pipelines Airflow, banco PostgreSQL, dashboards Streamlit e chat híbrido com DataFrame Agent e RAG.

Airflow PostgreSQL LangGraph Chroma Gemini API

Aplicação completa para ingestão, persistência e exploração de indicadores econômicos brasileiros. O projeto reúne pipelines ETL, banco relacional, dashboards analíticos e um chat com IA para consulta guiada de séries históricas, indicadores e documentação técnica do próprio sistema.

O que foi implementado

  • Pipelines Airflow: backfill histórico e atualização diária consumindo BCB, IPEA, SIDRA, Banco Mundial e CoinGecko.
  • Banco PostgreSQL: modelagem com tabelas de indicadores, valores históricos, logs de ETL, alertas e views para snapshot macroeconômico.
  • Dashboards Streamlit: uma versão executiva conectada ao banco e uma versão standalone em Plotly para demonstração.
  • Chat econômico híbrido: modo DataFrame Agent com pandas + Gemini para exploração tabular e modo RAG com LangGraph, Chroma e embeddings HuggingFace para responder com base no código, esquema e documentação.
  • Documentação indexada: arquivos de estrutura, overview e dashboard integrados à camada RAG para permitir respostas contextualizadas.

Fluxo end-to-end

  • Catálogo de indicadores definido em YAML para padronizar origem, categoria e frequência.
  • Extração e transformação idempotentes com cálculo de variações, médias e normalizações.
  • Consumo dos dados por dashboards e pelo chat de IA em diferentes modos de análise.

Conecta-Zap (Idosos)

Backend conversacional para trilha automatizada de letramento digital via WhatsApp, com onboarding, 10 pílulas educativas, agendamento recorrente, métricas operacionais e geração de dicas extras com IA.

FastAPI Evolution API WhatsApp Cloud Gemini PostgreSQL

Aplicação desenvolvida para automatizar uma jornada educativa e social no WhatsApp voltada a idosos, com foco em autonomia digital, segurança contra golpes e uso de serviços essenciais no celular. O sistema envia uma sequência de 10 pílulas de conteúdo com imagem e texto, registra interações, coleta respostas de CTA e feedback final, gera dicas extras com IA e disponibiliza métricas administrativas para acompanhamento da operação.

Problema resolvido

O projeto entrega conteúdo educativo de forma escalável e acessível dentro de um canal já adotado pelo público-alvo, sem exigir download de um novo app ou navegação por uma interface web tradicional.

O que foi implementado

  • Backend assíncrono com FastAPI: ciclo de vida completo, health check, arquivos estáticos e configuração centralizada por variáveis de ambiente.
  • Integração com WhatsApp: suporte a Evolution API e WhatsApp Cloud API, webhook, verificação, tratamento de texto, botões, listas, polls, imagem e áudio.
  • Fluxo conversacional: onboarding com coleta de nome e idade, opt-in/opt-out, controle de estados por usuário, CTAs por pílula, pedido de dicas extras e feedback final.
  • Conteúdo educacional estruturado: 10 pílulas sobre acessibilidade, segurança digital, golpes, PIX, Gov.br e Meu SUS Digital, com linguagem simplificada e apoio visual.
  • Agendamento automático: APScheduler com horários recorrentes, proteção contra reenvio duplicado e progressão automática da trilha.
  • Persistência e observabilidade: PostgreSQL assíncrono, exportação JSON/CSV, logs estruturados, correlation ID, Langfuse e proteção básica de PII.
  • Dicas extras com IA: Gemini via LangChain com timeout, fallback e prompt desenhado para linguagem simples e apropriada para idosos.

IEEE-CIS Fraud - MLOps

Pipeline MLOps ponta a ponta para fraude com simulação financeira, monitoramento contínuo de drift, retreinamento condicionado e comparação entre estratégias de operação sobre a base IEEE-CIS Fraud Detection.

MLOps AWS SageMaker Evidently MLflow LightGBM

Este projeto demonstra, ponta a ponta, como um pipeline MLOps completo opera sobre a base pública IEEE-CIS Fraud Detection, cobrindo ingestão, engenharia de features, treinamento, simulação financeira, monitoramento de drift e retreinamento condicionado com rastreabilidade operacional.

Destaques do pipeline

  • Fluxo orquestrado: ingestão, features, treino, simulação, monitoramento e retreinamento champion/challenger.
  • Modelagem: LightGBM, HistGradientBoosting e meta logistic stacking com features históricas por entidade.
  • Monitoramento: relatórios Evidently em HTML/JSON com PSI, KS, entropia do score e gatilhos de drift.
  • Rastreabilidade: MLflow para runs completos, parquets e CSVs para auditoria e exportação para dashboards.
  • Simulação de negócio: comparação entre Train-Once, Q30, Q3M e estratégia baseada em drift, considerando custos de compute e retenção pós falso positivo.

Resultado de destaque

Com os thresholds atuais, a estratégia acionada por drift entregou aproximadamente US$ 559 mil de lucro acumulado, contra cerca de US$ 229 mil no cenário Train-Once, preservando rastreabilidade financeira e operacional do pipeline.
